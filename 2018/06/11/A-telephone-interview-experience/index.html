记一次电话面试经历

发表于 2018-06-11 | 分类于 面试

第一次电话面试，面的是杭州下沙一家公司，有点紧张，而且感觉自己学的还太浅了，对自己的过往项目经历也没有好好梳理，面试官很专业，感觉懂得很多。那么就谨以此文来记录一下这次电面经历。

电话面试（一）18min

1. 你能大概介绍下你对 Laravel 有什么见解吗，或者说比其他框架的优势

数据库迁移，多对多关系（模型），API转换层，开发比较方便。

反思：还有路由和artisan命令行方便，ORM数据操作方便，社区活跃，框架所用思想比较现代化。

2. 你平常除了搞 PHP 和 Java 还学过其他什么语言

Python做过一些爬虫，TensorFlow做过简单人工智能，Go学过基础语法，Swoole做过一些websocket，看了一下workman，主要项目前端都是Vue，用Ionic做了简单小论坛。

3. 你做的最大的项目中遇到的困难和难点，你又是怎么解决的

讲了大二时候做二手书店处理复杂的表间关系，门禁系统websocket多进程共享变量的问题解决。以前git的冲突的处理。

反思：缺少对自己现有项目的总结和梳理，回答的不够好。

4. 用过什么消息中间件之类的。（不过问我的应该是会什么技术栈）和编程有关的一些。

数据库：MySql，Oracle，Redis。跟七牛云、FTP之类的对接。JWT。easywechat。

反思：简历中没写好自己的技术栈。

5. 在使用 Redis 的时候常用的操作

get 取数据，del 删数据，设置数据有效时间

反思：Redis用的还是太少。过段时间要好好看下手册。

6. 大概说下 Redis 支持哪些数据类型

因为当时做项目只用了string所以没回答全

反思：string，hash，list，set，sort

7. 手机上打开一个浏览器，访问www.baidu.com，会经历什么

先查hosts表，有就指向DNS服务器，访问CDN服务器，返回html信息，浏览器接收到显示出来。

反思：以前看过这个，但印象不是很深了， 从输入URL到页面加载发生了什么

8. session 和 cookie 有什么区别

session在服务端存，cookie再客户端存，session对服务端压力比较大一点。

9. session 和 cookie 有什么联系吗

回答了存cookie会再服务端对应有session的id

反思：都是会话的一种方式。目前大多数的应用都是用 Cookie 实现Session跟踪的。第一次创建Session时，服务端会通过在HTTP协议中反馈到客户端，需要在 Cookie 中记录一个Session ID，以便今后每次请求时都可分辨你是谁。

10. https相关的东西

443 端口，公私玥验证，非对称数据加密

11. 了解过哪些数据加密算法

RSA，(MD5，SHA256 这些不是），Argon2

反思：Bcrypt 和 Argon2
